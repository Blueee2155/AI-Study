# 注册失败问题修复指南

## 🔍 问题诊断

您遇到的"注册失败,请重试"错误是由于以下原因造成的:

### 根本原因
1. **前端代码已修改但未重新编译**
   - 我修改了 `src/services/api.ts` 中的 `baseURL` 从空字符串改为 `'http://127.0.0.1:8000'`
   - 但由于系统中没有安装 Node.js,无法运行 `npm run build` 重新编译前端代码
   - 当前 `dist/` 目录中的 JS 文件仍然是旧版本,API 请求发送到错误的地址

2. **CORS 预检请求问题**
   - 浏览器在发送跨域 POST 请求前会先发送 OPTIONS 预检请求
   - 如果前端 base URL 配置不正确,会导致 CORS 失败

## ✅ 解决方案

### 方案一:安装 Node.js 并重新编译(推荐)

#### 步骤 1: 下载并安装 Node.js
1. 访问 [Node.js 官网](https://nodejs.org/)
2. 下载 LTS(长期支持)版本
3. 运行安装程序,使用默认设置即可

#### 步骤 2: 验证安装
```bash
node --version
npm --version
```

#### 步骤 3: 重新编译前端
```bash
cd "d:\AI Study\ai-tutor-frontend"
npm install
npm run build
```

#### 步骤 4: 重启前端服务
```bash
python start-server.py
```

现在访问 http://localhost:3000/register 应该可以正常注册了!

---

### 方案二:手动修改编译后的文件(临时方案)

如果您暂时无法安装 Node.js,可以手动修改 dist 目录中的 JS 文件:

#### 步骤 1: 找到编译后的 API 文件
打开 `d:\AI Study\ai-tutor-frontend\dist\assets\index-D7Zi9yYE.js` (文件名可能不同)

#### 步骤 2: 搜索并替换 baseURL
在文件中搜索 `baseURL:""` 或 `baseURL:''`,将其替换为 `baseURL:"http://127.0.0.1:8000"`

⚠️ **注意**: 这种方法不推荐,因为文件是压缩过的,容易出错。

---

### 方案三:使用 Vite 开发服务器(最佳开发体验)

Vite 开发服务器自带热重载和代理功能,无需手动编译:

#### 步骤 1: 安装 Node.js(同方案一)

#### 步骤 2: 启动 Vite 开发服务器
```bash
cd "d:\AI Study\ai-tutor-frontend"
npm run dev
```

这会自动在端口 5173 启动开发服务器,并且已经配置了 API 代理(见 `vite.config.ts`)。

#### 步骤 3: 访问应用
打开 http://localhost:5173

## ❓ 常见问题

### Q: 为什么之前能注册,现在不行了?
A: 之前使用的是 Vite 开发服务器(带代理),现在使用的是 Python 静态文件服务器(无代理)。修改 api.ts 后需要重新编译才能生效。

### Q: 我可以继续使用当前的静态服务器吗?
A: 可以,但必须先重新编译前端代码,或者手动修改 dist 中的 JS 文件。

### Q: Node.js 会影响现有环境吗?
A: 不会。Node.js 是独立安装的,不会影响 Python 或其他工具。
