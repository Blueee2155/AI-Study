# TRAE Security Review

**TRAE Security Review** 是一款专注于代码变更安全审计的自动化能力（Skill），以语义级理解为核心，旨在高效、精准地识别增量代码中引入的安全风险，并以结构化、可视化的方式提供反馈。

本能力为技术使用者设计，尤其适用于在IDE中进行自动化安全卡点，或作为安全工程师、开发人员在代码审查（Code Review）阶段的得力助手。

## 核心能力与特性

- **🎯 变更驱动，聚焦增量风险**：严格遵循“不翻旧账”原则，仅对当前变更引入或修改的代码进行安全分析，有效避免历史遗留问题的干扰。
- **🔇 高信噪比，减少干扰**：内置明确的扫描排除规则，默认忽略如拒绝服务（DoS）、资源耗尽、缺少速率限制、纯粹的编码风格以及测试代码等问题，将注意力集中在具有真实攻击路径和高影响力的漏洞上。
- **📊 可视化分析，洞察变更意图**：自动将代码变更的关键逻辑提炼成 Mermaid 流程图或序列图，无论是业务流程的变动还是技术实现的调整，都能一目了然，极大地提升了代码变更的可理解性。
- **📝 结构化输出，清晰明确**：审查结果以清晰的表格形式呈现，内容包括问题标题、修复建议以及指向问题代码的精确链接，方便快速定位与修复。
- **⚖️ 置信度评估，辅助决策**：对发现的每个问题提供置信度评分。对于不确定的发现，会通过降低置信度分数来提示，以便进行二次人工确认或交由后续流程处理。


## 工作流程

本能力遵循一套严谨的、分阶段的执行流程，以确保审查的全面性与准确性。

1.  **确定审查范围**
    -   首先会识别出需要审查的代码范围，这通常是工作区中尚未提交的变更（`git diff`）或合并请求中的代码差异。

2.  **上下文收集**
    -   在进行任何分析之前，会调用相关工具获取与代码变更相关的完整上下文，包括但不限于被修改文件的原始内容、项目的设计文档等。这是确保分析准确性的基础。

3.  **推断作者意图**
    -   通过对代码变更模式的整体分析（例如，是在增加错误处理、重构变量名，还是修改配置），提炼出本次变更最核心的“作者意图”。这个意图的摘要（如：“意图：为 `process_user` 方法添加空指针检查”）将作为后续所有步骤的关键背景信息。

4.  **可视化摘要生成**
    -   这是本能力的核心特色之一。它会**始终**生成至少一张 Mermaid 图表来总结关键变更。
        -   对于简单的变更，会生成一张图。
        -   对于涉及多个模块或业务与技术双重改动的复杂变更，会生成两张图（一张业务流程图，一张技术流程图）。
    -   图表优先采用**流程图（flowchart）**和**序列图（sequenceDiagram）**，以动态展示逻辑链、调用顺序或数据流。静态的、仅做分类的框图是被明确反对的。
    -   图表会使用高对比度的颜色和文字来高亮变更部分，确保在任何主题下都清晰可读。
