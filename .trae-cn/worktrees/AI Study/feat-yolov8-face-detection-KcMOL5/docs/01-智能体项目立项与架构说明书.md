## 一、项目概述

### 1.1 项目背景

考研竞争日益激烈，备考过程漫长且枯燥，学习者普遍面临两个核心痛点：

1. **学习状态难以维持**：长时间自习容易走神、打瞌睡，缺乏外部监督机制
2. **疑难问题即时解答**：复习中遇到知识点疑惑时，缺乏专业、及时的辅导资源

本项目旨在构建一款 Web 端 AI 智能学习助手，通过 **视觉感知 + 大模型推理** 的 Agent 架构，实现"学习状态监督 + 考研智能问答"双重功能，帮助考研学子提升学习效率。

### 1.2 项目目标

| 目标层级 | 描述 | 验收指标 |
|---------|------|---------|
| 核心目标1 | 实时学习状态监督 | 摄像头检测人脸位置与状态，识别"专注/分心/开小差/离开"四种状态，准确率 ≥ 85% |
| 核心目标2 | 考研智能问答 | 集成 DeepSeek 大模型 + RAG 知识库，流式回答考研问题，首字延迟 ≤ 2 秒 |
| 辅助目标1 | 温柔语音提醒 | 虚拟宠物每 25 分钟语音提醒休息，分心时即时提示 |
| 辅助目标2 | 学习数据记录 | 记录专注时长、分心次数，生成每日学习报告 |

### 1.3 智能体定位

本系统是一个**多模态学习监督智能体**，具备以下 Agent 能力：

- **感知（Perception）**：通过摄像头捕获用户面部图像，经 OpenCV/YOLOv8 检测人脸位置与状态
- **规划（Planning）**：根据视觉感知结果 + 学习计时器状态，规划提醒策略（是否提醒、提醒内容、提醒方式）
- **行动（Action）**：通过虚拟宠物动画 + Web Speech 语音合成执行提醒；通过大模型 + RAG 执行知识问答

## 二、对标人群

本产品的目标用户主要涵盖以下几类人群，不同人群的使用场景和核心诉求各有侧重：

| 人群类型 | 典型特征 | 核心诉求 | 使用频率 |
|---------|---------|---------|---------|
| 在校考研学生 | 大三、大四在校生，有固定自习场所，备考周期 6-12 个月 | 学习状态监督、知识点答疑、复习规划建议 | 每天 4-8 小时 |
| 在职考研人群 | 工作 1-3 年，利用下班后和周末时间备考，时间碎片化 | 高效利用碎片时间、疑难问题快速解答、学习进度管理 | 每天 2-4 小时，周末全天 |
| 二战/三战考生 | 有过考研经历，有一定基础，需要针对性提升薄弱科目 | 薄弱点突破、真题深度解析、心理状态调节 | 每天 6-10 小时 |
| 自考/考证人群 | 准备各类职业资格考试、自学考试的学习者 | 知识点问答、学习状态监督 | 按需使用 |
| 远程学习学生 | 上网课、线上学习的中学生或大学生 | 课堂专注力监督、作业答疑 | 每天 3-6 小时 |

其中，在校考研学生和在职考研人群是核心目标用户，占预计总用户量的 70% 以上。产品功能设计围绕考研场景展开，同时兼顾其他自主学习场景的需求。

## 五、Agent 智能体设计

### 4.1 Agent 核心理念

本系统的智能体遵循 **感知-规划-行动（Perception-Planning-Action）** 循环：

```
        ┌─────────────────────────────────┐
        │         Agent 核心循环           │
        │                                 │
        │   ┌─────────┐                   │
        │   │  感知    │ ← 摄像头图像采集   │
        │   │Perception│ ← OpenCV 人脸检测  │
        │   └────┬────┘ ← YOLOv8 状态分类  │
        │        │                        │
        │        ▼                        │
        │   ┌─────────┐                   │
        │   │  规划    │ ← 状态判定规则     │
        │   │Planning  │ ← 计时器状态融合   │
        │   └────┬────┘ ← 提醒策略决策     │
        │        │                        │
        │        ▼                        │
        │   ┌─────────┐                   │
        │   │  行动    │ → 语音提醒输出     │
        │   │ Action   │ → 宠物动画触发     │
        │   └────┬────┘ → 大模型问答响应   │
        │        │                        │
        │        └─────── 循环 ──────────┐│
        │                               ││
        └───────────────────────────────┘│
                                        │
        ┌───────────────────────────────┘
        │
        ▼
   (下一次感知周期)
```

### 4.2 感知层（Perception）

| 感知通道 | 数据来源 | 处理方式 | 输出 |
|---------|---------|---------|------|
| 视觉感知 | 浏览器摄像头帧 (Base64 JPEG) | OpenCV Haar Cascade 多分类器融合检测 | 人脸框坐标、置信度 |
| 状态感知 | 人脸位置与面积比 | 规则引擎计算 (中心偏移、面积占比、宽高比) | 专注/分心/开小差/离开 |
| 时间感知 | 前端计时器 | 25 分钟周期监测 | 是否到达提醒时间 |
| 文本感知 | 用户输入的问题 | DeepSeek API 向量化 + RAG 检索 | 相关知识文档 |

**视觉感知算法流程**：

1. 前端每 500ms 截取摄像头帧，压缩为 416px 宽 JPEG，Base64 编码
2. 后端解码图像，转灰度图，CLAHE 对比度增强
3. 加载多个 Haar Cascade 分类器（frontalface_default/alt2/alt/alt_tree）
4. 多尺度检测（scaleFactor: 1.03~1.05, minSize: 20~30px）
5. IoU 合并重叠框（阈值 0.3），保留最大人脸
6. 计算人脸中心偏移量、面积占比，输出状态判定

### 4.3 规划层（Planning）

**状态判定规则**（优先级从高到低）：

| 条件 | 状态 | 专注分数 | 原因 |
|------|------|---------|------|
| 无人脸或面积比 < 1.5% | 离开 | 10 | 人脸太小或离屏幕太远 |
| 中心偏移 X > 62% 或 Y > 72% | 开小差 | ≤ 32 | 人脸偏离画面中心 |
| 检测置信度 < 0.4 | 分心 | ≤ 55 | 检测置信度偏低 |
| 中心偏移 X < 50% 且 Y < 62% 且面积 ≥ 2% | 专注 | ≥ 78 | 检测到稳定人脸 |
| 其他情况 | 分心 | ≤ 65 | 人脸位置不稳定 |

**提醒策略规划**：

| 触发条件 | 提醒类型 | 提醒内容 |
|---------|---------|---------|
| 25 分钟计时到达 | 休息提醒 | "你已经学习25分钟啦，起来走动一下吧~" |
| 检测到"分心"状态 | 分心提醒 | "注意力好像飘走了哦，深呼吸重新集中一下吧~" |
| 检测到"开小差"状态 | 开小差提醒 | "好像走神了呢，要继续学习吗？" |
| 检测到"离开"状态超 30 秒 | 离开提醒 | "你去哪里了呀？学习还没结束哦~" |

### 4.4 行动层（Action）

| 行动类型 | 执行方式 | 技术实现 |
|---------|---------|---------|
| 语音提醒 | Web Speech API 合成中文语音 | `SpeechSynthesisUtterance`, `lang='zh-CN'`, `rate=0.9` |
| 宠物动画 | CSS 动画切换宠物表情与动作 | 专注→安静看书；分心→敲屏幕；离开→张望 |
| 人脸框绘制 | Canvas 实时绘制检测框与状态标签 | 绿色框(专注) / 橙色框(开小差) / 红色遮罩(分心) |
| 知识问答 | DeepSeek API 流式生成回答 | SSE 推送，逐字显示，末尾附知识总结 |

### 4.5 问答 Agent 工作流

```
用户提问
    │
    ▼
┌─────────────────┐
│ 1. 问题向量化    │  ← DeepSeek embedding API
│ embedding(提问) │  ← 失败则返回模拟向量
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 2. RAG 知识检索  │  ← pgvector 余弦相似度 (生产)
│ search_knowledge│  ← SQLite LIKE 关键词搜索 (开发回退)
│ top_k=5         │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 3. 构建 Prompt  │  ← 科目专属系统提示
│ system + history│  ← 注入检索到的参考资料
│ + question      │  ← 追加知识总结要求
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 4. DeepSeek API │  ← httpx 原生流式调用
│ stream=True     │  ← 逐字 yield 给 SSE
│ max_tokens=4096 │  ← temperature=0.7
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 5. 保存对话历史  │  ← 用户消息 + AI回复存入 chat_history
│ commit to DB    │  ← session_id 关联会话
└─────────────────┘
```

## 七、安全设计

### 6.1 认证与授权

| 安全措施 | 实现方式 |
|---------|---------|
| 密码存储 | bcrypt 哈希 (passlib) |
| Token 签发 | JWT HS256, 有效期 24 小时 |
| Token 验证 | 每个受保护端点通过 `get_current_user` 依赖注入验证 |
| API Key 保护 | 运行时内存存储，不写入文件；通过 settings API 动态配置 |

### 6.2 数据安全

| 安全措施 | 实现方式 |
|---------|---------|
| SQL 注入防护 | SQLAlchemy ORM 参数化查询 |
| CORS 配置 | 默认允许所有源 (开发)，生产环境限制具体域名 |
| 代理环境变量清除 | 启动时清除 HTTP_PROXY/HTTPS_PROXY 等，避免 API 调用泄露 |
| 摄像头隐私 | 图像仅在内存中处理检测，不持久化存储用户面部图像 |

## 九、亮点与差异点

与市面上现有的学习类 App、刷题软件和 AI 问答工具相比，本产品的核心差异和亮点主要体现在以下几个方面：

### 9.1 视觉监督 + 知识问答二合一

现有产品大多只做单一方向：要么专注学习计时/番茄钟（如 Forest、番茄 ToDo），要么专注 AI 问答（如各类 GPT 工具）。本产品将**实时视觉状态监督**和**考研领域知识问答**深度结合，形成"监督 + 辅导"的完整学习闭环——既能盯着你学，又能帮你解决不会的问题。

### 9.2 摄像头本地感知，保护隐私

人脸检测完全在用户本地浏览器 + 后端服务器内存中完成，**不存储任何用户面部图像**，检测完即丢弃。相比需要上传人脸数据到云端的产品，隐私风险更低，用户心理负担更小。

### 9.3 轻量级部署，零成本起步

- 前端单 HTML 文件即可运行，无需安装客户端
- 后端基于 FastAPI + SQLite，开发环境零配置启动
- 大模型使用 DeepSeek API，按调用量付费，学生也能用得起
- 支持 Docker Compose 一键部署到自己的服务器

### 9.4 考研垂直领域优化

通用 AI 问答工具虽然功能强大，但缺乏考研场景的针对性。本产品：
- 按政治 / 英语 / 数学 / 专业课四个方向定制系统提示词
- 内置 RAG 知识库，可以导入考研资料做精准检索
- 回答自动附带知识总结（核心要点、关键公式、记忆技巧、常见考点）
- 语气和内容风格贴合考研学生需求，更像一位陪考学长/学姐

### 9.5 温柔陪伴式设计

不同于冷冰冰的打卡和监督工具，本产品通过虚拟宠物 + 温柔语音提醒的方式，营造陪伴感：
- 分心时温柔提醒，而非刺耳警告
- 25 分钟到点提醒休息，符合番茄工作法
- 宠物表情随学习状态变化，增加趣味
- 学习报告以鼓励为主，减少焦虑感
