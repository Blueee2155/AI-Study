# Folio Template

## 1. Meta

- type: folio
- description: Traditional single-column document — medium information density
- layout: single-column, centered
- max-width: 900px

***

## 2. Design Rules

### Allowed

- Use `--color-primary` (mapped to `--accent`) **only** for: links, `border-bottom` on H1, `border-bottom` on H2, strong emphasis within key callouts
- Use `--color-secondary` (mapped to `--accent2`) **only** for: table header backgrounds, code block backgrounds, blockquote left-border
- Hierarchy established solely through **font-size + font-weight + one divider strategy + whitespace**
- Line-height: titles 1.3–1.4, body text 1.7–1.8 (generous for readability)
- Tables: full-width, `1px solid var(--rule)` borders, zebra-striped rows using `var(--bg2)`
- Code/pre blocks: `var(--bg2)` background + `1px solid var(--rule)` border, `border-radius: 4px`
- Blockquotes: `4px solid var(--accent2)` left border + `var(--bg2)` background
- Choose exactly one divider strategy per output: either `h2` uses `border-bottom`, or sections are separated by `<hr>`; never use both in the same document
- Only `var(--font)` for body/heading; `var(--font-mono)` for code and pre blocks only
- Vertical rhythm: consistent spacing (16px paragraph, 20–40px between sections)

### Forbidden

- No `box-shadow`, no `filter: blur()`, no gradients of any kind
- No `border-radius` > 4px anywhere
- No card-based layouts or grid dashboards — this is a traditional flowing document
- No hardcoded hex colors — everything via `var(--*)`
- No decorative icons, emojis, or illustrations in headings
- No multi-column body text (single reading column only)
- No sticky/fixed positioning
- No colored section backgrounds to distinguish chapters
- No ASCII art

***


## 5. Components

All component names below are descriptive examples. Labels, captions, and nearby headings may be renamed to fit the document's actual structure.

### Full-Width Table (zebra-striped)

```html
<div class="table-wrap">
  <table>
    <thead>
      <tr>
        <th>{col_1}</th>
        <th>{col_2}</th>
        <th>{col_3}</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>{val_1}</td>
        <td>{val_2}</td>
        <td>{val_3}</td>
      </tr>
      <tr>
        <td>{val_1}</td>
        <td>{val_2}</td>
        <td>{val_3}</td>
      </tr>
    </tbody>
  </table>
</div>
```

### Code Block

```html
<pre><code>{code_content}</code></pre>
```

### Blockquote

```html
<blockquote>
  <p>{quote_text}</p>
```


```html

  <li>{item_3}</li>

```

### Unordered List

```html
<ul>
  <li>{item_1}</li>
```

### Nested List

```html

  <li>{parent_item}

      <li>{child_item_1}</li>

    </ul>
  </li>

```

### Multi-Column Grid (`.col-grid`)

```html
<div class="col-grid">
  <div class="col-item">
    <h4>{col_title_1}</h4>
    <p>{col_content_1}</p>
  </div>
  <div class="col-item">
    <h4>{col_title_2}</h4>
    <p>{col_content_2}</p>
  </div>
  <div class="col-item">
    <h4>{col_title_3}</h4>
    <p>{col_content_3}</p>
  </div>
</div>
```

### Chart Figure (`.chart-figure`)

Every ECharts chart MUST be wrapped in a `<figure>` with a visible `<figcaption>` title.

```html
<figure class="chart-figure">
  <figcaption>{chart_title}</figcaption>
  <div id="chart-{unique_name}" style="width:100%;min-height:400px"></div>
</figure>
```

### Diagram Figure (`.diagram`)

Every diagram (Mermaid, PlantUML, Graphviz, inline SVG) MUST be wrapped in a `<figure>` with a `<figcaption>` title.

```html
<figure class="diagram">
  <pre class="mermaid">
    flowchart LR
      A --> B --> C
  </pre>
  <figcaption>Figure X: {diagram_title}</figcaption>
</figure>
```
