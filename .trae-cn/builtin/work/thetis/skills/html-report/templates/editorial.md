- type: editorial
- description: Editorial long-form layout
- layout: single-column, centered
- max-width: 860px

***

## 2. Design Rules

### Allowed

- Use `--color-primary` (mapped to `--accent`) **only** for accent: links, border-left on blockquotes, kicker top-border, active TOC marker
- Use `--color-secondary` (mapped to `--accent2`) **only** for subtle hover states and tag backgrounds
- Hairline `1px solid var(--rule)` borders as dividers; prefer controlled-length (≤80%) rather than full-width
- Hierarchy is established solely through **font-size + font-weight + whitespace**
- Line-height: titles 1.15–1.3, body text 1.55–1.65
- Tags: solid `var(--bg2)` background + `1px solid var(--rule)` border, no radius > 4px
- Only `var(--font)` for body/heading; `var(--font-mono)` for code only
- Vertical rhythm: consistent spacing multiples (0.5rem, 1rem, 1.5rem, 2rem, 3rem)

### Forbidden

- No `box-shadow`, no `filter: blur()`, no gradients (`linear-gradient`, `radial-gradient`) except subtle quote background
- No `border-radius` ≥ 8px anywhere
- No color-based hierarchy (no colored section backgrounds to distinguish chapters)
- No hardcoded hex colors or font-family names in CSS — everything via `var(--*)`
- No decorative icons or emojis in headings
- No multi-column body text (single reading column only)

## 5. Components

Component names in this section are suggestive rather than mandatory. You may rename headings, labels, and surrounding section titles to match the report structure.

### Blockquote (`.quote`)

```html
<div class="quote">
  <p>{quote_text}</p>
  <cite>— {author}, {source}</cite>
</div>
```

### Data Table

```html
<div class="table-wrap">
  <table>
    <thead>
      <tr>
        <th>{col_1}</th>
        <th>{col_2}</th>
        <th>{col_3}</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>{val_1}</td>
        <td>{val_2}</td>
        <td>{val_3}</td>
      </tr>
    </tbody>
  </table>
</div>
```

### Callout / Sidebar (`.callout`)

```html
<div class="callout">
  <div class="callout-title">{callout_label}</div>
  <p>{callout_content}</p>
</div>
```

### Tag Row (`.tag-row` + `.tag`) — OPTIONAL

```html
<!-- Optional: only include when explicit keyword tagging adds value (e.g. taxonomy, search labels). Do NOT add a tag row just to echo the section's key points — the body text already conveys them. -->
<div class="tag-row">
  <span class="tag">{tag_text}</span>
  <!-- repeat as needed -->
</div>
```

### Meta Card (`.meta-card`)

```html
<div class="meta-card">
  <div class="label">{meta_label}</div>
  <div class="value">{meta_value}</div>
</div>
```

### Metric Row (`.metric-row`)

```html
<!-- Flexible: use any number of metrics, grid auto-fits -->
<div class="metric-row">
  <div class="metric">
    <div class="number">{number_value}</div>
    <div class="desc">{metric_description}</div>
  </div>
  <!-- repeat as needed -->
</div>
```

### Two-Column Layout (`.two-col`)

```html
<div class="two-col">
  <div>
    <h3>{left_heading}</h3>
    <p>{left_content}</p>
  </div>
  <div>
    <h3>{right_heading}</h3>
    <p>{right_content}</p>
  </div>
</div>
```

### Chart Figure (`.chart-figure`)

Every ECharts chart MUST be wrapped in a `<figure>` with a visible `<figcaption>` title.

```html
<figure class="chart-figure">
  <figcaption>{chart_title}</figcaption>
  <div id="chart-{unique_name}" style="width:100%;min-height:400px"></div>
</figure>
```

### Diagram Figure (`.diagram`)

Every diagram (Mermaid, PlantUML, Graphviz, inline SVG) MUST be wrapped in a `<figure>` with a `<figcaption>` title.

```html
<figure class="diagram">
  <pre class="mermaid">
    flowchart LR
      A --> B --> C
  </pre>
  <figcaption>Figure X: {diagram_title}</figcaption>
</figure>
```
