# Cockpit Template

## 1. Meta

- type: cockpit
- description: Data visualization dashboard with KPI cards, chart grids, and interactive analytics — high information density
- layout: single-column with multi-column grids
- max-width: 1200px

***

## 2. Design Rules

### Allowed

- Use `--accent` for: KPI delta indicators, chart accent colors, active tab markers, primary highlights
- Use `--accent2` for: card hover states, secondary chart colors, soft label backgrounds, subtle emphasis
- Cards: `var(--bg2)` background + `1px solid var(--rule)` border + `border-radius: 10px`
- Glass-style subtle elevation: cards can use very subtle background opacity differences, but NO backdrop-filter or blur
- Grid layouts: 4-col for KPIs, 2-col or 3-col for chart panels, flexible for content
- Chart containers: consistent padding, min-height for aspect ratio
- Sparklines: inline SVG, stroke using accent colors
- Monospace font for data values, labels, axis ticks
- Responsive grid (collapses gracefully)
- Hairline `1px solid var(--rule)` borders for cards/dividers
- Hierarchy through **font-size + font-weight + whitespace + grid layout**
- Line-height: titles 1.1–1.2, body text 1.5–1.6
- Deltas use `--accent` for upward trends, `--muted` for downward trends
- Inline SVG sparklines with `stroke-width: 1.5` and no fill for clean data density
- Monospace font (`var(--font-mono)`) for: KPI values, axis labels, table numbers, delta indicators, breadcrumb, footer

### Forbidden

- No `backdrop-filter` or `filter: blur()` — must work in headless renderers
- No `position: sticky` or `position: fixed` — this is a static report document
- No live animation (`setInterval`, CSS animation loops) — exception: one-shot count-up animation for KPI numbers on page load is allowed
- No full-width header that spans the viewport — header is inside the max-width container
- No external CDN or network resources (except ECharts which is inlined)
- No hardcoded hex colors for theme-dependent elements
- No gradients for backgrounds (subtle chart area fills are OK)
- No `box-shadow` — elevation is conveyed by border + background only
- No multi-column body text wider than the chart panel
- No tables inside multi-column layouts (`.insight-grid`, `.chart-grid.cols-2`, or any custom two-column grid) — tables with 4+ columns MUST use full-width single-column layout

***

## 5. Components

Component names below describe common dashboard use cases, not fixed semantics. Titles, labels, badges, legends, and section headings can all be renamed to fit the narrative.

### KPI Card

```html
<div class="kpi-card">
  <div class="kpi-label">{label}</div>
  <div class="kpi-value" data-count-to="{numeric_value}" data-prefix="{prefix}" data-suffix="{suffix}">{value}</div>
  <div class="kpi-delta positive">↑ {delta_value} ({delta_pct})</div>
  <div class="kpi-footnote">{comparison_period}</div>
</div>
```

- `data-count-to`: target numeric value (e.g. `2400000`, `4.8`, `84200`)
- `data-prefix`: text before number (e.g. `$`, ``)
- `data-suffix`: text after number (e.g. `%`, `M`, `pp`)
- Fallback: if JS disabled, `{value}` shows the final formatted value

Delta variants:

```html
<!-- Positive -->
<div class="kpi-delta positive">↑ 12.4%</div>
<!-- Negative -->
<div class="kpi-delta negative">↓ 3.1%</div>
<!-- Neutral -->
<div class="kpi-delta neutral">— 0.0%</div>
```

### KPI Count-Up Animation

Place this script once before `</body>`, after all ECharts scripts:

```html
<script>
  (function() {
    var duration = 1200;
    var els = document.querySelectorAll('[data-count-to]');
    function easeOut(t) { return 1 - Math.pow(1 - t, 3); }
    function formatNumber(n, decimals) {
      return n.toLocaleString(undefined, {
        minimumFractionDigits: decimals,
        maximumFractionDigits: decimals
      });
    }
    function animate(el) {
      var target = parseFloat(el.getAttribute('data-count-to'));
      var prefix = el.getAttribute('data-prefix') || '';
      var suffix = el.getAttribute('data-suffix') || '';
      var decimals = (String(target).split('.')[1] || '').length;
      var start = performance.now();
      function step(now) {
        var progress = Math.min((now - start) / duration, 1);
        var value = target * easeOut(progress);
        el.textContent = prefix + formatNumber(value, decimals) + suffix;
        if (progress < 1) requestAnimationFrame(step);
      }
      requestAnimationFrame(step);
    }
    if ('IntersectionObserver' in window) {
      var observer = new IntersectionObserver(function(entries) {
        entries.forEach(function(entry) {
          if (entry.isIntersecting) {
            animate(entry.target);
            observer.unobserve(entry.target);
          }
        });
      }, { threshold: 0.3 });
      els.forEach(function(el) { observer.observe(el); });
    } else {
      els.forEach(animate);
    }
  })();
</script>
```

### Sparkline Row

Uses inline SVG sparkline from `diagrams/svg/sparkline.md`.

```html
<div class="sparkline-row">
  <span class="spark-label">{metric_name}</span>
  <span class="spark-value">{current_value}</span>
  <svg class="sparkline" viewBox="0 0 100 24" preserveAspectRatio="none">
    <polyline class="primary" points="{x1},{y1} {x2},{y2} {x3},{y3} ..." />
  </svg>
  <span class="spark-delta positive">↑ {delta}</span>
</div>
```

**IMPORTANT: Sparkline data consistency rule**

The polyline `points` MUST reflect the actual trend implied by the delta:
- If delta is **positive** (↑), the sparkline should trend **upward** (later y values should be lower — SVG y-axis is inverted)
- If delta is **negative** (↓), the sparkline should trend **downward** (later y values should be higher)
- The sparkline shape should be a plausible mini time-series, not random squiggles

Example — upward trend (delta ↑ 12%):
```html
<polyline class="primary" points="0,20 20,18 40,16 60,14 80,10 100,6" />
```

Example — downward trend (delta ↓ 3%):
```html
<polyline class="primary" points="0,8 20,10 40,12 60,14 80,17 100,20" />
```

Example — flat/stable (delta — 0%):
```html
<polyline class="primary" points="0,12 20,11 40,13 60,12 80,11 100,12" />
```
