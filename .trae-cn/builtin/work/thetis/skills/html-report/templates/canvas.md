# Canvas Template

## 1. Meta

- type: canvas
- description: Notion-like workspace document layout
- layout: single-column, centered
- max-width: 920px

***

## 2. Design Rules

### Allowed

- Notion-like workspace feel: airy single-column reading flow, restrained chrome, soft dividers, modular information blocks
- Use `--accent` for: links, decision callout border-left, pill highlights, avatar gradients
- Use `--accent-soft` for: decision callout background, status pill backgrounds
- Cards/panels: `var(--bg)` background + `1px solid var(--line)` border + `border-radius: 10px`
- Hairline `1px solid var(--line)` borders for dividers and card edges
- Hierarchy through **font-size + font-weight + whitespace**
- Line-height: titles 1.1–1.2, body text 1.55–1.65
- Display font (`var(--display)`) for h1, h2; system font (`var(--body)`) for body text
- Monospace font (`var(--mono)`) for: labels, timestamps, pill text, meta strong labels — uppercase + letter-spacing for metadata
- Avatar circles: gradient backgrounds with white text initials
- Grid layouts: 2-column for panels, single-column for lists

### Forbidden

- No `box-shadow` or `filter: blur()` — elevation via border + background only
- No `position: sticky` or `position: fixed` — this is a static document
- No live animation (`setInterval`, CSS animation loops)
- No external CDN or network resources
- No hardcoded hex colors for theme-dependent elements
- No gradients for backgrounds (small avatar decorative gradients are OK)
- No decorative emojis in headings
- No interactive buttons or controls (no "Refresh", "Auto" toggles, etc.)

## 5. Components

Components are **all optional** and can be mixed freely within any `<section>`. Repeat sections as needed.

Component names below are only suggested patterns. Section titles, block titles, list labels, and status wording can all be renamed to match the note-taking context.

### Task / Checklist

```html
<div class="agenda">
  <!-- Dynamic: repeat N items, add class="done" for completed -->
  <div class="agenda-item done">
    <div class="check">✓</div>
    <div class="body"><strong>{item_title}</strong><small>{item_description}</small></div>
    <div class="time">{time_info}</div>
  </div>
  <div class="agenda-item">
    <div class="check"></div>
    <div class="body"><strong>{item_title}</strong><small>{item_description}</small></div>
    <div class="time">{time_info}</div>
  </div>
  <!-- ... add or remove items freely -->
</div>
```

### Callout / Summary Block

```html
<div class="callout">
  <h3>{callout_title}</h3>
  <ul>
    <!-- Dynamic: repeat N items -->
    <li><strong>{key_point}</strong> {detail_text}</li>
  </ul>
</div>
```

Or with paragraphs:

```html
<div class="callout">
  <h3>{callout_title}</h3>
  <p>{paragraph_text}</p>
</div>
```

### Data Table (scrollable both directions)

```html
<div class="table-wrap">
  <table>
    <thead>
      <tr>
        <!-- Dynamic: repeat N columns -->
        <th>{col_header}</th>
      </tr>
    </thead>
    <tbody>
      <!-- Dynamic: repeat N rows -->
      <tr>
        <td>{value}</td>
        <td class="owner"><span class="av" style="background:linear-gradient(135deg,{c1},{c2})">{initials}</span>{name}</td>
        <td>{date}</td>
        <td><span class="pill pill-{status}">{status_text}</span></td>
      </tr>
    </tbody>
  </table>
</div>
```

Pill status variants: `pill-todo`, `pill-progress`, `pill-blocked`, `pill-done`, `pill-warn`

### Grid Panels (2-column)

```html
<div class="grid">
  <!-- Dynamic: repeat N panels (renders in 2-column grid) -->
  <div class="panel">
    <h3>{panel_title}</h3>
    <p>{panel_content}</p>
    <!-- Optional: key-value list -->
    <div class="kv-list">
      <div><strong>{label}</strong>{value}</div>
      <div><strong>{label}</strong>{value}</div>
    </div>
  </div>
</div>
```

### KPI Row

```html
<!-- Use .cols-N class: cols-2, cols-3, cols-4, cols-5 -->
<div class="kpi-row cols-{N}">
  <!-- Dynamic: repeat N cards -->
  <div class="kpi-card">
    <div class="kpi-label">{label}</div>
    <div class="kpi-value">{value}<small>{unit}</small></div>
    <div class="kpi-delta positive">{delta_text}</div>
  </div>
</div>
```

Delta variants:
```html
<div class="kpi-delta positive">↑ 6 vs last week</div>
<div class="kpi-delta negative">↓ 2 vs last week</div>
<div class="kpi-delta neutral">· Stable</div>
```

### Activity / Update Feed

```html

  <!-- Dynamic: repeat N items -->
  <div class="activity-item">
    <span class="av" style="background:linear-gradient(135deg,{c1},{c2})">{initials}</span>
```



### Chart Figure (`.chart-figure`)

Every ECharts chart MUST be wrapped in a `<figure>` with a visible `<figcaption>` title.

```html
<figure class="chart-figure">
  <figcaption>{chart_title}</figcaption>
  <div id="chart-{unique_name}" style="width:100%;min-height:400px"></div>
</figure>
```

### Diagram Figure (`.diagram`)

Every diagram (Mermaid, PlantUML, Graphviz, inline SVG) MUST be wrapped in a `<figure>` with a `<figcaption>` title.

```html
<figure class="diagram">
  <pre class="mermaid">
    flowchart LR
      A --> B --> C
  </pre>
  <figcaption>Figure X: {diagram_title}</figcaption>
</figure>
```
