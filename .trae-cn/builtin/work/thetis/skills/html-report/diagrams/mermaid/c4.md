# C4 Diagram

Use for system architecture at different abstraction levels: Context (L1), Container (L2), Component (L3), and Dynamic views.

## Context Diagram (Level 1)

Shows the system in scope and its relationships with users and external systems.

```mermaid
C4Context
    title System Context Diagram - Online Banking

    Person(customer, "Banking Customer", "A customer with one or more bank accounts")
    Person(admin, "Bank Admin", "Internal bank staff managing accounts")

    System(banking, "Online Banking System", "Allows customers to view and manage their bank accounts")

    System_Ext(email, "Email System", "SendGrid-based email notifications")
    System_Ext(mainframe, "Mainframe Banking", "Core banking system holding account data")
    System_Ext(atm, "ATM Network", "External ATM cash withdrawal network")

    Rel(customer, banking, "Views accounts, makes payments", "HTTPS")
    Rel(admin, banking, "Manages customer accounts", "HTTPS")
    Rel(banking, email, "Sends notifications", "SMTP")
    Rel(banking, mainframe, "Gets account info, makes payments", "XML/HTTPS")
    Rel(customer, atm, "Withdraws cash")

    UpdateRelStyle(customer, banking, $offsetY="-40")
    UpdateRelStyle(banking, mainframe, $offsetX="-60")
```

## Container Diagram (Level 2)

Zooms into the system boundary, showing containers (applications, databases, etc.).

```mermaid
C4Container
    title Container Diagram - Online Banking

    Person(customer, "Customer", "Banking customer")

    System_Boundary(banking, "Online Banking System") {
        Container(spa, "Single-Page App", "React, TypeScript", "Provides banking functionality via browser")
        Container(mobile, "Mobile App", "React Native", "Provides banking functionality via mobile")
        Container(api, "API Gateway", "Go, Hertz", "Routes and authenticates API requests")
        Container(accounts, "Accounts Service", "Go", "Manages account data and transactions")
        Container(notifications, "Notification Service", "Python", "Sends emails and push notifications")
        ContainerDb(db, "Database", "PostgreSQL", "Stores account data, transactions")
        ContainerQueue(mq, "Message Queue", "RabbitMQ", "Async message routing")
    }

    System_Ext(mainframe, "Mainframe", "Core banking")
    System_Ext(email, "Email Service", "SendGrid")

    Rel(customer, spa, "Uses", "HTTPS")
    Rel(customer, mobile, "Uses", "HTTPS")
    Rel(spa, api, "Calls", "JSON/HTTPS")
    Rel(mobile, api, "Calls", "JSON/HTTPS")
    Rel(api, accounts, "Routes to", "gRPC")
    Rel(accounts, db, "Reads/Writes", "SQL")
    Rel(accounts, mq, "Publishes events", "AMQP")
    Rel(mq, notifications, "Consumes", "AMQP")
    Rel(notifications, email, "Sends via", "SMTP")
    Rel(accounts, mainframe, "Syncs", "XML/HTTPS")
```

## Component Diagram (Level 3)

Zooms into a single container, showing internal components.

```mermaid
C4Component
    title Component Diagram - API Gateway

    Container_Boundary(api, "API Gateway") {
        Component(router, "Router", "Go, Chi", "HTTP request routing and middleware")
        Component(auth, "Auth Middleware", "Go", "JWT validation and user extraction")
        Component(ratelimit, "Rate Limiter", "Go", "Token bucket rate limiting per user")
        Component(circuit, "Circuit Breaker", "Go", "Prevents cascade failures")
        Component(transform, "Request Transformer", "Go", "Maps REST to gRPC")
        Component(logger, "Access Logger", "Go", "Structured request logging")
    }

    Container_Ext(spa, "SPA", "React")
    Container_Ext(accounts, "Accounts Service", "Go")
    Container_Ext(redis, "Redis", "Cache + rate limit state")

    Rel(spa, router, "HTTPS requests")
    Rel(router, auth, "Delegates auth check")
    Rel(router, ratelimit, "Checks rate limit")
    Rel(ratelimit, redis, "Reads/updates counters")
    Rel(router, circuit, "Wraps outbound calls")
    Rel(circuit, transform, "On success")
    Rel(transform, accounts, "gRPC call")
    Rel(router, logger, "Logs all requests")
```

## Dynamic Diagram

Shows interactions for a specific scenario/use case.

```mermaid
C4Dynamic
    title Login Flow

    ContainerDb(db, "Database", "PostgreSQL")
    Container(api, "API Server", "Go")
    Container(cache, "Cache", "Redis")
    Container(spa, "Web App", "React")

    Rel(spa, api, "1. POST /login (email, password)", "HTTPS")
    Rel(api, db, "2. Query user by email", "SQL")
    Rel(api, api, "3. Verify password hash")
    Rel(api, cache, "4. Store session", "SET")
    Rel(api, spa, "5. Return JWT + refresh token", "HTTPS")
```

## C4 Elements Reference

| Element | Syntax | Description |
|---------|--------|-------------|
| Person | `Person(id, "Name", "Desc")` | Human user |
| System | `System(id, "Name", "Desc")` | System in scope |
| External System | `System_Ext(id, "Name", "Desc")` | External system |
| Container | `Container(id, "Name", "Tech", "Desc")` | Application/service |
| Container DB | `ContainerDb(id, "Name", "Tech", "Desc")` | Database |
| Container Queue | `ContainerQueue(id, "Name", "Tech", "Desc")` | Message queue |
| Component | `Component(id, "Name", "Tech", "Desc")` | Internal component |
| Boundary | `System_Boundary(id, "Name") { }` | System boundary |
| Container Boundary | `Container_Boundary(id, "Name") { }` | Container boundary |
| Relationship | `Rel(from, to, "Label", "Tech")` | Directed relationship |
| Back Relationship | `BiRel(from, to, "Label", "Tech")` | Bidirectional |

## Styling

```
UpdateElementStyle(elementId, $fontColor="red", $bgColor="blue", $borderColor="black")
UpdateRelStyle(from, to, $textColor="blue", $lineColor="blue", $offsetX="10", $offsetY="-20")
UpdateLayoutConfig($c4ShapeInRow="3", $c4BoundaryInRow="1")
```

## Best Practices

1. **Start at Context (L1)** — always create context diagram first
2. **One diagram per level** — don't mix levels in the same diagram
3. **Clear boundaries** — use `System_Boundary` and `Container_Boundary`
4. **Label technology** — include tech stack in container/component descriptions
5. **Direction on relationships** — use verb phrases ("Calls", "Reads from", "Sends to")
6. **Include protocols** — show communication protocols (HTTPS, gRPC, AMQP, SQL)
7. **External systems** — always distinguish internal vs external with `_Ext` suffix
8. **Audience-appropriate** — Context for stakeholders, Container for architects, Component for developers
